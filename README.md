## Overview
This repository contains the analysis and evaluation of a web application to identify common security vulnerabilities. The report examines web headers, cookies, input validation, and server configurations to determine potential risks and provide recommendations. The purpose of this analysis is to demonstrate web security testing techniques and provide actionable insights for improving application security.  

---

## Contents
The PDF report includes:  

- **Executive Summary** – Overview of the web application assessment, identified vulnerabilities, and mitigation measures.  
- **Introduction** – Explanation of web application security, common risks, and testing objectives.  
- **Scope of the Report** – Description of the analysis approach, limitations, and environment setup.  
- **Tools and Methodology** – Security tools used (OWASP ZAP, Nmap, Browser Developer Tools) and step-by-step workflow.  
- **Findings & Observations** – Detailed examination of the application including:  
  - Missing or misconfigured security headers  
  - Cookies without proper security flags  
  - Exposed server and service information  
  - Input validation weaknesses  
- **Risk Classification** – Severity levels and potential impacts of identified issues.  
- **Recommendations** – Suggested security improvements and best practices.  
- **Conclusion** – Final findings and actionable insights to enhance web application security.
